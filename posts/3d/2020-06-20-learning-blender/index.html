Particle Properties

粒子属性： 基于一个对象，然后把它复制到另一个对象的表面上。

点击 ➕ 后，就会新生成一个粒子属性。一个粒子属性分为两类，Emitter 动态类粒子，Hair 静态类粒子，我们根据情况选择其中一个就好了。

Hair

选择 hair 后，物体表面全是发射状的细线，为了使用特定的对象作为粒子。

  1. 选择 Render 下拉框，Render As 属性设置为 Object 。(Path 是用于做毛发之类的粒子)
  2. 展开 Render 下的 object 子属性，Instance Object 属性设置为 <作为粒子的对象>。

以上就是 静态粒子 的最基本的使用方法。

Rotation

勾选 Hair 类例子下面的 Advanced ，下面就会出现 Rotation 属性面板。

  1. Orientation Axis 属性设置。[Normal, X, Y, Z，甜甜圈酥皮上的糖屑选择 Normal]
  2. Randomize 属性设置，各个粒子的旋转轴不一样。[酥皮糖屑稍微调整一下该值]
  3. Randomize Phase 属性设置，粒子绕着各自的法线轴旋转。[酥皮糖屑拉高一点该值]

以上就是 Rotation 的简单介绍。

Weight Paint

定义那些部分有粒子。

切换为 Weight Paint Mode 后，整个模型都会变成蓝色，用笔触绘画后，编程红色，红色表示1，蓝色表示0，值越大，粒子在该区域出现的概率越大。

打开 Vertex Groups 属性，将 Density 设置为 Group。

Texture

绘制贴图，可以让模型看起来更美观，更有质感，增加更多的细节。当然，看了 UE5 宣传预告后，贴图绘制方面可能会有所减少，因为在建模阶段，我们就可以直接将绝大多数的细节做出来。虽然 blender 里能直接绘制贴图，但是整个体验还不是很好，我们只能新建文件，将绘制信息存储在图片里，这样的话，非常不利于后续的调整，最好能引入 PS 里层的概念，将信息先存储在 blend 文件里，保存时，再将所有信息整合，写入图片文件里。

Diffuse

漫反射贴图。这是最基础的贴图，和小孩子画画一样，我们直接在模型上面画画，不过会受限于模型本身的精度，在三维空间里，除了基础的色彩，我们不能表达更多的东西了。比如你想要在模型上画一口井，可能在你的视角里看，有立体感，但是当你旋转视图，从其他视角观察时，你就只能看见一个平面，直接拉跨，所以我才说和小孩子画画一样。

Bump

凹凸贴图。那么为了解决上诉的痛点，我们就要使用 Bump 贴图了，不同于 Diffuse 贴图的是，Bump 贴图是 灰度图 是黑白电视机色调的图。

⚠NOTE：绘制任何贴图前，一定要先保存图片文件，防止后续不当操作以致绘制信息消失（因为你不保存的话，所有信息都是存储在 blend 文件缓冲区的）。

Physics

该功能用于模拟现实生活中的各种物理效果，比如苹果从树上掉到地上，blender 可以模拟这一过程。截至 2020.06.20 blender 2.83版本里，blender 内置了 8 种常见的物理属性。

Rigid Body [刚体]

=======

1. Rigid Body [刚体]

dev

要解释该功能的作用就有点绕了。在任何三维软件里，建模是最基本的工作，为模型添加 Rigid Body 是使用模型的一种方法。为什么会有为模型添加 Rigid Body 的需要呢？在游戏领域里有 Sprite (中译：灵魂) 的概念，在添加 Rigid Body 前，每一个模型都只是 Sprite，像动画里的灵魂一样，我们能看见，但是我们却不能同他们握手拥抱，因为我们会穿过他们，在三维软件里没添加 Rigid Body 的模型就是这样的，带来的结果就是，模型与模型之间无法产生接触，发生碰撞，在做游戏的时候这是一个非常令人头大的问题。

初次使用 Rigid Body 模拟苹果掉在地上需要注意的参数就只有上面三个了。

1. Type： 有 Active 和 Passive 之分，由于地面可以看作是静止不动的，所以这里改成 Passive，苹果会掉落，是运动的，这里设置为 Active。

2. Shape： 细心的话就会发现这是设置 Collisions Shape 的，想要得到最好的效果，不管是地面还是苹果，都设置为 Mesh更符合实际（物体是什么形状，它的碰撞范围就是什么样）。

3. Margin： 这个概念和网页里的 Margin 有点类似，通俗的解释就是在原来的碰撞检测范围外再添加一点间距，当然是间距越小效果越好了。为什么会有这个选项呢？当一个模型太薄的时候这个就派上用场了。

TIPS

Depth of Field

人眼有焦点距离，范围之内的物体比较清晰可见，范围外的物体会随着距离的增加而不可见。为了模拟该功能，Camera 下设置了 Depth of Field 属性。

Focus Object： 可以自由设置焦点的物体。

F-Stop： 设置上诉焦点距离。

USE

通常添加一个 Plain Axis 作为 Focus Object 这样我们可以在不影响场景的情况下调整焦点位置。
